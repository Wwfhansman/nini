# 项目总览

## 项目名称

妮妮 Kitchen Agent。

## 一句话定位

面向家庭厨房的一顿饭任务型 AI 终端，通过记忆、库存、视觉和状态机，陪用户完成从规划到烹饪再到复盘的一顿饭。

## 背景

原项目是智能语音厨房助手，技术重心在本地 STT/TTS、RAG、实时语音交互和打断检测。这个方向有工程价值，但在省赛人工智能应用赛道中容易被理解为“语音版菜谱问答”。新版项目需要把重点从底层语音工程转到产品体验和 Agent 应用闭环：

- 用户不是来问一个菜谱，而是在厨房里完成一顿饭。
- AI 不只是回答，而是要管理当前任务、记住家庭偏好、根据现实食材调整计划。
- 屏幕不是聊天窗口，而是厨房工作台。

## 核心目标

1. 做出一条可录制、可复现、可答辩的一顿饭闭环。
2. 让评委看到它不是菜谱推荐器，而是有状态、有记忆、有视觉、有工具调用的厨房 Agent。
3. 源码结构要像真实产品原型，支持后续扩展，而不是一次性演示工程。

## 核心用户场景

用户在厨房里准备晚饭，手上不方便频繁操作手机。用户面对的是家庭约束和现实变化：

- 自己最近减脂。
- 妈妈不吃辣。
- 家里库存有限。
- 实际食材数量和一开始想象不一致。
- 做饭过程中临时想调整口味。

妮妮需要承担：

- 规划今晚吃什么。
- 识别当前食材。
- 根据实际食材修正菜谱。
- 一步一步引导烹饪。
- 记录用户口味偏好。
- 完成后复盘和更新库存。

## 省赛版本边界

必须完成：

- 一顿饭闭环。
- 结构化 Agent 协议。
- 本地 P0 状态机。
- SQLite 记忆和库存。
- Recipe、Memory、Inventory、Vision 四类核心 skill。
- 临时测试页面。
- 正式前端终端界面。
- Mock/hybrid/real provider 切换。
- 演示脚本、架构文档、答辩材料。

优先完成：

- 火山 ASR/TTS 接入。
- 七牛 MaaS 对话、视觉、任务模型接入。
- 事件流或轮询式工具调用时间线。
- 家庭菜谱知识库轻量版本。

延期：

- 复杂实时语音打断算法。
- 大规模菜谱库。
- 完整多用户权限系统。
- 真实硬件终端。
- 商用品质视觉识别准确率。
- 插件市场式 skill 系统。

## 项目成败标准

成功标准：

- 演示视频中能清楚看到：现实食材打断计划、记忆立即改变步骤、P0 指令不走模型。
- 源码中能看到：状态机、schema 校验、SQLite 落库、provider 抽象、mock 兜底。
- PPT 中能讲清：结构化 Agent 控制 UI、状态机和 Agent 协同、记忆/库存/视觉共同驱动任务。

失败特征：

- 演示看起来像聊天框加菜谱推荐。
- 前端写死全部流程，后端只是传假数据。
- 记忆只是日志，不影响当前决策。
- 视觉只是上传图片展示结果，不改变菜谱和库存。
- 所有功能都调用大模型，没有本地确定性控制。
