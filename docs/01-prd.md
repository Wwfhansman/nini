# 需求文档 PRD

## 1. 产品定义

妮妮 Kitchen Agent 是一个家庭厨房 AI 终端原型。它以语音/文本为入口，以厨房工作台界面为核心，以视觉、记忆、库存和菜谱技能为支撑，帮助用户完成一顿饭。

## 2. 用户画像

### 主要用户

家庭厨房中的普通做饭用户。

特点：

- 做饭时双手被占用，不适合长时间操作手机。
- 需要同时考虑家人偏好、库存和烹饪步骤。
- 希望 AI 给出明确、短句、可执行的下一步。

### 省赛评委视角

评委关注：

- 是否是人工智能应用，而不是 API 拼接。
- 是否有清晰场景和用户价值。
- 是否有工程完整性。
- 是否能稳定演示。

## 3. 核心问题

家庭做饭不是简单的菜谱搜索，而是多约束任务：

- 谁吃？
- 有什么不能吃？
- 家里有什么食材？
- 实际食材是否足够？
- 当前做到哪一步？
- 临时偏好如何影响当前步骤？
- 做完后如何沉淀经验？

## 4. 核心能力

### 4.1 家庭约束理解

用户输入：

> 我最近减脂，妈妈不吃辣，冰箱里有鸡胸肉、番茄、鸡蛋，今晚做什么？

系统需要识别：

- 用户目标：减脂。
- 家庭成员约束：妈妈不吃辣。
- 库存：鸡胸肉、番茄、鸡蛋。
- 任务：规划今晚菜品。

输出：

- 写入 memory。
- 更新 inventory。
- 推荐适合约束的菜品。

### 4.2 菜品规划

系统根据 memory、inventory、recipe catalog 和用户输入规划菜品。

省赛固定主菜：

- 低脂不辣番茄鸡胸肉滑蛋。

规划页展示：

- 菜名。
- 推荐理由。
- 食材匹配程度。
- 预计时间。
- 是否符合偏好。

### 4.3 视觉修正

用户上传食材图或使用摄像头拍照。

系统输出结构化 observation：

- 番茄只有半个。
- 鸡胸肉偏少。
- 鸡蛋 2 个。

系统根据 observation：

- 修正库存。
- 调整菜谱份量。
- 降低番茄酸度。
- 增加鸡蛋比例。

### 4.4 烹饪工作台

进入 cooking 后，中间主工作台只显示当前最重要动作：

- 第几步/共几步。
- 当前动作。
- 用料。
- 火候。
- 计时器。
- 下一步按钮。

不展示长篇菜谱，不让用户在厨房里阅读大段文字。

### 4.5 本地控制指令

这些 P0 指令不调用大模型：

- 开始。
- 下一步。
- 上一步。
- 暂停。
- 继续。
- 完成。

后端状态机负责更新 `terminal_state`。

### 4.6 长期记忆即时生效

用户输入：

> 记住我不喜欢太酸。

系统需要：

- 写入长期记忆。
- 当前菜谱立即调整。
- 烹饪工作台显示改动标记。

示例改动：

- 番茄翻炒时间从 90 秒降到 45 秒。
- 鸡蛋比例增加。
- 不额外加醋。

### 4.7 复盘

完成后展示：

- 总用时。
- 库存扣减。
- 本次写入的记忆。
- 下次建议。

示例：

> 已记录：用户不喜欢太酸。下次做番茄类菜品时默认减少番茄用量，并优先使用鸡蛋或豆腐中和酸味。

### 4.8 家庭菜谱知识库

知识库不是主角，是 recipe knowledge skill。

首版支持：

- 文本/Markdown 菜谱导入。
- 家庭菜谱命中展示。
- 规划时借鉴家庭菜谱。

示例：

用户导入：

> 妈妈版番茄炒蛋：不放辣，鸡蛋多一点，出锅前加一点糖。

后续规划时：

> 命中家庭菜谱：妈妈版番茄炒蛋。当前方案沿用“不放辣、鸡蛋多一点”，但考虑减脂，不额外加糖。

## 5. 非功能需求

### 稳定性

- 演示必须支持 mock 模式。
- 模型输出必须经过 schema 校验。
- provider 失败要有回退。

### 可解释性

- 右侧显示工具调用时间线。
- 显示命中的 memory、inventory、recipe knowledge。
- 显示哪些步骤是状态机处理，哪些是 Agent 处理。

### 可扩展性

- provider 可切换。
- skill 模块边界清晰。
- 数据按 `terminal_id` 隔离。

### 可答辩性

- 明确说明当前是原型系统。
- 不宣称完全自主智能体。
- 不宣称商用准确率。

## 6. 优先级

### P0

- `/api/chat`
- `/api/control`
- `/api/state`
- SQLite memories、inventory_items、terminal_state、tool_events。
- Recipe/memory/inventory skills。
- Mock agent 和 mock vision。
- 临时测试页面。

### P1

- 七牛 MaaS real LLM provider。
- Vision real provider。
- TTS 接入。
- ASR 接入。
- 正式前端终端界面。

### P2

- 家庭菜谱知识库。
- 事件流。
- Memory compaction。
- 更丰富 recipe catalog。

### P3

- 向量检索。
- 多用户账号系统。
- 真实硬件部署。
