# 演示脚本

## 演示目标

视频和答辩现场要让评委看到三件事：

1. 现实食材会打断原计划，系统会调整。
2. 用户长期记忆不是日志，会立即改变当前步骤。
3. 高频控制指令由本地状态机处理，不依赖大模型。

## 演示时长建议

3 到 5 分钟。

## 演示准备

### 固定样例食材图

图片内容建议：

- 半个番茄。
- 少量鸡胸肉。
- 2 个鸡蛋。

Mock vision 返回：

```json
{
  "ingredients": [
    {"name": "番茄", "amount": "半个", "confidence": 0.91},
    {"name": "鸡胸肉", "amount": "少量", "confidence": 0.86},
    {"name": "鸡蛋", "amount": "2个", "confidence": 0.94}
  ],
  "notes": ["番茄数量不足原计划", "鸡胸肉适合做一人份"]
}
```

### 初始数据

空或少量默认数据：

- terminal_id: `demo-kitchen-001`
- user: `我`
- household: `张家厨房`

## 演示流程

### 1. 开场

打开终端页面。

讲解口径：

> 家庭做饭不是简单搜索菜谱，而是多约束任务。今天的场景是：我减脂，妈妈不吃辣，家里食材不完整，还希望 20 分钟内做完。

界面：

- 中间大标题：今晚做一顿适合家人的饭。
- 右侧：家庭记忆、库存摘要为空或较少。

### 2. 输入家庭约束

用户输入：

> 我最近减脂，妈妈不吃辣，冰箱里有鸡胸肉、番茄、鸡蛋，今晚做什么？

预期系统动作：

- `memory_write`: 用户最近减脂。
- `memory_write`: 妈妈不吃辣。
- `inventory_update`: 鸡胸肉、番茄、鸡蛋。
- `recipe_plan`: 规划菜品。

主工作台进入 `planning`。

展示内容：

- 菜名：低脂不辣番茄鸡胸肉滑蛋。
- 推荐理由：
  - 符合减脂目标。
  - 不放辣。
  - 使用现有食材。
  - 预计 18 分钟。

### 3. 视觉修正

用户上传固定食材图。

讲解口径：

> 真实厨房里，计划经常被现实食材打断。这里视觉发现番茄只有半个，鸡胸肉也偏少。

预期系统动作：

- `vision_observe`: 识别食材。
- `inventory_update`: 修正库存数量。
- `recipe_adjust`: 调整菜谱。

主工作台短暂进入 `vision`，然后回到 `planning`。

展示调整：

- 改成一人份。
- 番茄用量减少。
- 鸡蛋比例增加。
- 酸味降低。
- 鸡胸肉切薄片，保证熟得快。

### 4. 开始烹饪

用户点击或输入：

> 开始做。

预期系统动作：

- `local_control: start_cooking`
- 不调用模型。

主工作台进入 `cooking`。

展示：

- 第 1/6 步：鸡胸肉切薄片并轻腌。
- 用料：鸡胸肉、少量盐、淀粉。
- 火候：暂不加热。
- 计时器：03:00。
- 下一步按钮。

### 5. 记忆即时影响当前步骤

用户输入：

> 记住我不喜欢太酸。

预期系统动作：

- `memory_write`: 用户不喜欢太酸。
- `recipe_adjust`: 当前菜谱降酸。

主工作台仍保持 `cooking`。

显示 toast：

> 已写入长期记忆：用户不喜欢太酸。

当前步骤改动标记：

- 番茄翻炒时间 90 秒 -> 45 秒。
- 鸡蛋比例增加。
- 不额外加醋。

讲解口径：

> 这不是把偏好存成日志，而是立即影响当前烹饪流程。

### 6. 本地状态机控制

用户点击：

- 下一步。
- 暂停。
- 继续。

右侧时间线显示：

```text
local_control: next_step，未调用模型
local_control: pause_timer，未调用模型
local_control: resume_timer，未调用模型
```

讲解口径：

> 下一步、暂停、继续是确定性指令，由后端状态机处理，避免大模型延迟和误判。

### 7. 完成和复盘

用户点击：

> 完成。

主工作台进入 `review`。

展示：

- 总用时：示例 16 分钟。
- 库存扣减：
  - 鸡蛋 -2。
  - 番茄 -半个。
  - 鸡胸肉 -少量。
- 新增记忆：
  - 用户不喜欢太酸。
  - 妈妈不吃辣。
  - 用户最近减脂。
- 下次建议：
  - 番茄类菜品默认减少酸味。
  - 优先用鸡蛋或豆腐中和酸度。
  - 不推荐辣味做法。

### 8. 记忆闭环补问

用户输入：

> 下次做番茄类菜要注意什么？

预期回答：

> 我会默认降低酸度，减少番茄翻炒时间，增加鸡蛋或豆腐来中和口感，并继续保持不辣。

### 9. 导出家庭记忆卡片

点击导出。

展示 Markdown：

```md
# 张家厨房记忆卡

- 用户最近在减脂
- 用户不喜欢太酸
- 妈妈不吃辣
- 番茄类菜品默认降低酸度
```

## 失败兜底

- 模型 API 不稳定：切 `DEMO_MODE=mock`。
- 视觉识别不稳定：使用固定 mock vision。
- 语音失败：使用文本输入和 TTS 可选。
- 前端联调失败：用临时测试页面录制后端闭环。
